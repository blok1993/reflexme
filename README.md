# Predictor

> «Утром — прогноз. Вечером — проверка.»

Персональный психологический ассистент. Не гороскоп, не магия — система, которая учится на конкретном человеке и со временем делает прогнозы всё точнее.

---

## Суть продукта

Каждый день проходит по одному и тому же циклу:

**Утро (30 секунд)** → пользователь описывает своё состояние: настроение (1–5), фокус дня (работа, люди, энергия, эмоции, напряжение, контроль, отдых, фокус на себе) и несколько слов контекста. ИИ сразу генерирует три карточки прогноза.

**День** → пользователь живёт, зная что его ждёт.

**Вечер** → приложение возвращает те же три карточки и просит оценить каждую: сбылось / частично / мимо. Можно добавить короткий комментарий.

Из этой петли рождается главный эффект: **азарт проверки точности**. Пользователь начинает возвращаться каждый день, чтобы посмотреть — насколько точно приложение его «угадало».

---

## Три карточки прогноза

Каждый день ИИ генерирует именно три карточки — не больше, не меньше:

1. **Что вероятно** — один психологический или межличностный сценарий дня. Конкретный, не абстрактный. Не «возможны перемены», а «тебя будет задевать тон, а не слова».

2. **Твоя сила** — одна реальная точка силы или стратегия. Практичная рекомендация на сегодня.

3. **Ловушка дня** — одна когнитивная или эмоциональная ловушка, в которую этот человек может попасть именно сегодня.

Плюс **тип дня** — поэтичное название: «День внутреннего шума», «День тихой силы», «День ложной тревоги». Задаёт общий тон.

## Персонализация

Приложение персонализируется на нескольких уровнях:

### Онбординг
При первом входе собирается:
- **Пол** — влияет на грамматические формы в прогнозах (мужской/женский род)
- **Тон общения** — мягко / нейтрально / прямо. Меняет стиль формулировок
- **Имя** — необязательно, для приветствия
- **Дата рождения** — необязательно, используется как тихий контекст для LLM без явного упоминания

### Краткосрочная история (7 ревью)
При каждой генерации прогноза ИИ видит последние 7 завершённых дней: дата, настроение, фокус, точность прогноза, комментарий пользователя, тип дня. Это позволяет учитывать недавние паттерны.

### Долгосрочный профиль (profileSummary)
После 5+ ревью раз в 6–7 дней асинхронно генерируется текстовый профиль пользователя (~500–1000 символов). ИИ анализирует последние 30 дней и составляет психологическое резюме: типичные триггеры, точки силы, паттерны точности, характерные слова. Этот профиль вшивается в system prompt при каждой генерации прогноза. Чем больше данных — тем точнее формулировки.

## Идентификация пользователя

Приложение использует анонимную сессию. При первом открытии в браузере генерируется UUID v4, который сохраняется в localStorage и передаётся как заголовок `X-Device-ID` в каждом запросе. Бэкенд находит или создаёт пользователя по этому ID. Данные привязаны к устройству — смена браузера или устройства = новый пользователь.

## Экраны и навигация

### Умный редирект
При открытии `/` приложение определяет статус текущего дня и сразу ведёт пользователя туда, где он должен быть:
- Нет чек-ина → `/checkin`
- Есть чек-ин, нет прогноза → `/prediction` (идёт генерация)
- Утро + прогноз есть → `/prediction`
- Вечер + прогноз есть, ревью нет → `/review`
- Ревью завершено → `/insights`

Навигация снизу: **Сегодня** (умный редирект) | **Паттерны** (`/insights`) | **Профиль** (`/settings`)

### `/onboarding`
Три шага: приветствие → как это работает → настройка (пол, имя, тон, дата рождения). Защищён: если онбординг уже пройден, перенаправляет на `/checkin`.

### `/checkin`
Форма с тремя элементами: слайдер настроения (5 кнопок с emoji), выбор фокуса (8 пилюль), текстовое поле контекста (необязательно, макс 300 символов). Кнопка активна только после выбора настроения и фокуса. При выборе настроения фон приложения плавно меняет цвет (mood color system). Если чек-ин на сегодня уже есть — сразу редирект на `/prediction`. При 409 (чек-ин уже существует) — тоже редирект на `/prediction`.

### `/prediction`
Сначала показывает LoadingScreen с живыми фразами («Читаю твой день...», «Ищу паттерны...»). После загрузки: тип дня крупно + badge уверенности (Уверен / Вероятно / Приблизительно) + три карточки с stagger-анимацией. Кнопка «Поделиться» — текстовый шаринг через Web Share API. Вечером появляется кнопка «Проверить прогноз». Если прогноз не удалось сгенерировать — показывает кнопку «Попробовать ещё раз».

### `/review`
Показывает те же три карточки с кнопками оценки под каждой (Да / Частично / Мимо). Кнопка «Завершить день» активна только после оценки всех трёх. После сабмита — экран завершения с анимацией, процентом попадания и «Увидимся завтра утром». Через 3.5 секунды — редирект на `/insights`.

### `/insights` — три вкладки

**Неделя** — сводка за текущую неделю: totalDays, средняя точность, самый частый тип дня. Паттерны — текстовые инсайты, сгенерированные LLM по итогам недели. Список дней с цветными точками точности.

**История** — список всех дней. Каждый элемент кликабелен → переход на `/history/:date` где показан полный прогноз и оценка.

**Профиль** — три компонента:
- *Кривая точности* — SVG-спарклайн по всем завершённым дням. Показывает тренд: «↑ Прогнозы становятся точнее» / «→ Стабильная» / «↓ Снизилась».
- *Открытия* — паттерн-карточки. Появляются после 7 ревью. ИИ генерирует 3–5 нетривиальных психологических инсайтов. Ключевой критерий: «удивит ли это человека — скажет ли он "ого, не замечал"». Обновляются раз в 10 дней.
- *Мой язык* — облако слов из всех контекстных текстов чек-инов. Алгоритмически, без LLM. Размер = частота.

### `/settings`
Имя, тон общения, информация о приложении.

### `/history/:date`
Полная карточка дня: прогноз + оценки пользователя + комментарий. Доступна из вкладки История.

## Петля удержания

Приложение бьёт в несколько психологических механик одновременно:

**Эффект сериала** — у каждого дня есть начало (утренний прогноз) и конец (вечерняя проверка). Формирует привычку возвращаться дважды.

**Эффект азарта** — пользователь хочет проверить, насколько точно угадало. Это не пассивное чтение — это игра.

**Эффект соавторства** — оценивая прогнозы, пользователь обучает систему. Ощущение, что он создаёт что-то своё.

**Эффект зеркала** — кривая точности растёт. Словарь появляется. Открытия удивляют. Пользователь видит доказательство что приложение его изучило.

**Эффект узнавания** — прогнозы говорят его языком, используют его формулировки. Человек узнаёт себя в тексте.

## Что приложение НЕ является

- Не гороскоп и не астрология — прогнозы основаны только на данных самого пользователя
- Не психотерапия — «психологические» формулировки без диагнозов и советов
- Не геймификация — нет баллов, достижений, рейтингов
- Не социальная сеть — нет друзей, сравнений, публичных данных

Тон продукта: умный, спокойный, слегка интроспективный. Как умный друг, который много наблюдает и мало говорит лишнего.

## Деплой в production

1. Сменить `provider = "sqlite"` на `provider = "postgresql"` в `schema.prisma`
2. Подключить managed PostgreSQL (Supabase / Neon free tier)
3. Frontend → Vercel или Netlify  
4. Backend → Railway или Render  
5. **Бэкенд:** `DATABASE_URL`, `OPENAI_API_KEY`, `FRONTEND_URL` (для CORS: один origin или **через запятую** несколько, напр. `https://reflexme.ru,https://reflexme-front.onrender.com`)
6. **Фронт (Vite):** при отдельном хосте API задать **`VITE_API_BASE_URL`**, иначе запросы пойдут на тот же домен что и SPA и получат HTML вместо JSON (ошибки вида `ApiError: HTTP 200`). Пример: `https://твой-api.onrender.com/api/v1`
7. **SPA и F5 / прямые URL (`/checkin`, `/onboarding`, …):** сервер должен отвечать **`index.html`** на любой путь (**rewrite**, не редирект на `/`).
   - **Netlify:** в сборке уже есть **`public/_redirects`** (`/*` → `/index.html` с 200).
   - **Vercel:** файл **`apps/frontend/vercel.json`** с rewrites.
   - **Render Static:** в панели сервиса → **Redirects / Rewrites** → правило: **Source** `/*`, **Destination** `/index.html`, **Action** **Rewrite**.
   - **Apache (shared hosting):** в сборке есть **`public/.htaccess`** — включи `AllowOverride`/mod_rewrite.
   - **Обход без доступа к серверу:** при сборке задать **`VITE_USE_HASH_ROUTER=true`** → адреса вида `https://site.ru/#/checkin` (браузер всегда запрашивает только `/`). Для продакшена предпочтительнее корректный rewrite.
